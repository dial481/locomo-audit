# Image-Dependent Questions

LoCoMo is a multimodal dataset. Conversations contain image shares with BLIP-generated captions. This document identifies which questions depend on image content and how each system handles them.

---

## How Images Appear in LoCoMo

Each conversation in `data/locomo10.json` contains turns where speakers share images. These turns include three fields:

| Field | Description |
|-------|-------------|
| `img_url` | URL(s) to the crawled image (hosted on reddit, flickr, wordpress, pexels, etc.) |
| `blip_caption` | Auto-generated caption from the [Salesforce BLIP](https://huggingface.co/Salesforce/blip-image-captioning-large) image captioning model |
| `query` | The search query used by [icrawler](https://icrawler.readthedocs.io/en/latest/) to find the image |

Source: `snap-research/locomo/README.MD`, line 17:

> "If the turn contains images, it also includes a link to the image `img_url`, caption generated by the BLIP model for the image `blip_caption` and the search query used by the third party module icrawler to retrieve the image."

The dataset does not include the images themselves (line 26):

> "We do not release the images. However, the web URLs, captions and search queries for the images are included in the dataset."

---

## Image Pipeline and the Caption Mismatch Problem

The original LoCoMo generation pipeline (`snap-research/locomo/generative_agents/`) works as follows:

1. ChatGPT generates conversations where agents "share photos" with bracketed captions like `[shares a photo of empanadas]`
2. The `icrawler` module uses Google Image Search to find real images matching the intended description
3. BLIP captions the crawled images to produce `blip_caption`
4. The `text` field is cleaned to remove the bracketed caption

This creates a potential disconnect: the **intended** image content (from ChatGPT) may differ from the **actual** crawled image (found via web search), and the **BLIP caption** describes the crawled image, not the intended content.

- Each conversation contains 30 to 149 image turns

## Question Categories

| Scope | Count (Evaluated) | Description |
|-------|-------------------|-------------|
| All image-evidence QAs | 607 | At least one evidence turn contains an image, but most answers come from conversation text |
| Truly image-dependent QAs | 32 | Golden answer contains substantive words found only in the BLIP caption, not anywhere in the conversation text |

The 32 truly image-dependent questions are those where answering correctly requires information available only through the BLIP caption. 14 additional image-dependent questions fall in category 5 (excluded from evaluation), for a total of 46.

The "Caption-Only Words" column shows the specific words from the golden answer that are found only via the BLIP caption.

Methodology: For each evaluated question (categories 1-4), the golden answer is tokenized and compared against (a) the BLIP caption words of image-bearing evidence turns and (b) the full set of text words across all turns in the same conversation, including observations. ## How Systems Handle Images

**No system processes actual images.** All 5 systems receive images as text via BLIP captions prepended to the message content.

The shared data loader at `EverMind-AI/EverMemOS/evaluation/src/core/loaders.py` (lines 221-226) prepends captions:

```python
if msg.get("img_url"):
    blip_caption = msg.get("blip_caption", "an image")
    content = f"[{speaker_name} shared an image: {blip_caption}] {content}"
```

This means every system receives text like:

```
[Audrey shared an image: a photo of a woman with a tattoo of a dog and
sunflowers] I know right? They mean the world to me. So much that I got
tattoos of them on my arm.
```

The original LoCoMo evaluation code uses the same approach:

```python
if 'blip_caption' in dialog:
    dialogs.append(dialog['speaker'] + ' said, "' + dialog['text'] + '"'
                   + ' and shared ' + dialog['blip_caption'])
```

Source: `snap-research/locomo/task_eval/gpt_utils.py`, lines 92-93, 184-185

No adapter (EverMemOS, Mem0, MemOS, MemU, or Zep) downloads images, uses vision models, or processes multimodal content. The text BLIP caption is the only image representation available to any system.

## Golden Answer vs. BLIP Caption Mismatch

At least 1 question has a golden answer that describes an image **different** from what the BLIP caption says:

| Question ID | Golden Answer | BLIP Caption | Issue |
|------------|---------------|--------------|-------|
| `locomo_4_qa169` | "a painting of Aragorn" | "a photo of a painting of a man smoking a cigarette" | Different subject (query was "aragorn painting lord of the rings") |

The human annotators based the golden answer on the **intended** image content from the conversation generation process, not the **actual** crawled image described by BLIP. No system can answer this question correctly from the available data because the BLIP caption describes a different image than the golden answer expects. `locomo_4_qa169` is a category 4 question that does not appear in the 32 truly image-dependent list above (the word "Aragorn" does not appear in the BLIP caption).

A systematic check of all 32 truly image-dependent questions confirmed that their BLIP captions are semantically consistent with their golden answers. The mismatch problem appears limited to questions outside the image-dependent set.

This represents a data quality issue inherited from the LoCoMo benchmark's image crawling pipeline, separate from the 99 score-corrupting errors documented in [AUDIT_REPORT.md](../AUDIT_REPORT.md).

## Summary

1. 39.4% of evaluated questions (607/1,540) reference evidence turns containing images, but most are answerable from conversation text alone.
2. 32 evaluated questions have golden answers containing substantive words found only in BLIP captions and not in any conversation text. This count is an upper bound; some matches may be coincidental.
3. All systems use only text BLIP captions. No vision models are involved.
4. All systems underperform on image-evidence questions, with accuracy drops of 0.8% to 8.8% on the broad set and larger drops on the 32 truly image-dependent questions.
5. At least 1 golden answer is unanswerable from the BLIP caption because the crawled image does not match the intended image. All 32 truly image-dependent questions have BLIP captions consistent with their golden answers.
