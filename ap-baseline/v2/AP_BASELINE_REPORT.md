# Adversarial Plausibility Baseline -- Judge Leniency Stress Test

## What This Is

A frontier LLM (Claude Opus 4.6) was given the LoCoMo answer key and asked to generate
the most plausible-sounding **wrong** answers it could for all 1,540 questions. These
intentionally incorrect answers were then scored by the exact same LLM judge used to
evaluate all 5 published memory systems.

**If the judge is functioning correctly, the adversarial baseline should score near 0%.**
Any score significantly above 0% represents answers where the judge cannot distinguish
crafted wrong answers from correct ones.

## Methodology

### Answer Generation

- **Model**: Claude Opus 4.6 (claude-opus-4-6)
- **Task**: Generate maximally plausible wrong answers for each question
- **Input**: Full answer key (question + golden answer for all 1,540 questions)
- **Output**: `ap_eval_results.json` -- same format as published eval_results.json

### Scoring

- **Judge model**: gpt-4o-mini (temperature=0)
- **Judge prompt**: Loaded at runtime from `../evaluation/config/prompts.yaml`
  - This is the **identical file** used by the original EverMemOS evaluation pipeline
  - Not copied, not paraphrased -- loaded from the same source
- **Runs**: 3 independent judge calls per question
- **Scoring**: Per-run accuracy averaged across runs (matching `compute_acc.py`)
- **Concurrency**: 10 (matching original LLMJudge semaphore)

## Results

Out of 1540 questions with intentionally wrong answers, the judge
